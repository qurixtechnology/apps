Secure Chat — verschlüsselter P2P-Videochat

Secure Chat baut eine direkte Verbindung zwischen zwei Browsern auf (Peer-to-Peer über WebRTC). Video, Ton und Textnachrichten laufen direkt von Gerät zu Gerät und sind dabei durch WebRTC Ende-zu-Ende verschlüsselt (DTLS/SRTP). Es gibt kein Konto, keine Installation und keinen qurix-Server, der die Inhalte sieht oder speichert.

So funktioniert ein Anruf

  • ⚡ Schnell: Gib die Kurz-ID deines Gegenübers ein (z. B. EAGLE-RIVER-77) und rufe an. Beide müssen online sein. Für den Verbindungsaufbau wird der öffentliche PeerJS-Signalserver genutzt — er vermittelt nur den Kontakt, überträgt aber keine Gesprächsinhalte.
  • 📴 Offline: Ganz ohne Signalserver. Erstelle eine Einladung (ein Textcode), schicke sie über einen beliebigen Kanal an dein Gegenüber, und füge dessen Antwort-Code wieder ein. Damit verbindet ihr euch auch ohne erreichbaren Server.

Identität & Verifizierung

Beim ersten Start erzeugt die App lokal ein kryptografisches Schlüsselpaar. Daraus wird deine gut merkbare Kurz-ID abgeleitet. Bei eingehenden Anrufen zeigt die App, ob der Anrufer ein bekannter, verifizierter Kontakt ist — oder warnt, wenn eine ID neu ist oder sich der Schlüssel eines bekannten Kontakts geändert hat.

Daten & Speicherung

Deine Identität und deine Kontakte werden ausschließlich in der lokalen localStorage deines Browsers gespeichert. Export with data erzeugt eine HTML-Kopie, die nur die Kontaktliste mitnimmt (Kurz-IDs, Namen, öffentliche Schlüssel) — dein privater Schlüssel bleibt immer auf diesem Gerät. Export blank liefert eine leere Kopie.

Hinweise

  • Kamera- und Mikrofonzugriff wird erst beim Start eines Anrufs abgefragt, nicht schon beim Öffnen der Seite. Auch ohne Kamerafreigabe funktioniert der Textchat.
  • Für den direkten Verbindungsaufbau wird ein öffentlicher STUN-Server (Google) verwendet.
  • Kein offizielles Sicherheitsaudit — für vertrauliche Hochsicherheits-Kommunikation ohne weitere Prüfung nicht geeignet.
